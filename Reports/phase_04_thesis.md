# Phase 4 Thesis: Text Emotion Model Exploration (Multiple Iterations)

## Strategic Intent
Phase 4 introduced the third and final sensory modality: text. The question was: "Can we classify the emotional content of written text, and which NLP architecture performs best?" Unlike face and voice, text emotion detection had no established model — four distinct scripts were written, each testing a different combination of model architecture, preprocessing strategy, and inference logic. This was the most iterative phase in the project, reflecting uncertainty about the right approach.

## Scope & Boundaries
Four scripts were developed in sequence: (1) `textemo.py` trained a basic BiLSTM from CSV data with no attention mechanism, producing a baseline model at `max_len=50`. (2) `textcalemo.py` tested loading and running a separately-trained CNN text model (`emotion_TEXT_cnn_model.h5`) with `max_len=100`, confirming that CNN architectures could also solve this task. (3) `textemotion.py` introduced the custom `AttentionLayer` for the first time and used glob-based dynamic file discovery for model artifacts, while importing but never using `PorterStemmer`. (4) `textemotion_tf212.py` added the `is_context_clear()` sentence filter and the `rewrite_sentence()` negation engine — a 183-line script representing the most sophisticated text processing attempt before the API transition.

## Failure Analysis
Phase 4 failed on three fronts. First, the `is_context_clear()` function was too restrictive: it rejected sentences with fewer than 3 words (eliminating "I'm sad"), sentences with more than 50% non-alphabetic characters (eliminating "I'm so happy!!!"), and sentences lacking explicit emotion keywords (eliminating "The sunset was beautiful"). Second, the `NEGATION_MAP` approach was fundamentally unscalable — mapping "not happy" → "sad" via regex cannot handle the infinite variety of English negation patterns ("I wouldn't say I'm particularly thrilled" has no regex match). Third, the per-sentence `model.predict()` loop in `textemotion_tf212.py` created compounding latency for paragraphs. These three issues were solved in Phase 8 by trusting the retrained BiLSTM+Attention model to handle negation natively, removing the `is_context_clear()` gate entirely, and implementing batch tensor prediction.
